# イベント当日：本番環境への切り替えマニュアル

このドキュメントは、イベント当日に向けて、現在蓄積されているテストデータをリセットし、参加者がクリーンな状態でアプリを利用できるようにするための手順書です。

## 概要

Firestore（データベース）には現在、テスト中に入力された位置情報や進捗データが蓄積されています。このままイベントを迎えると、「数日前に記録されたダミーの先頭集団」が表示されたままになるなどの不具合が発生します。

これを防ぐため、**「テストデータを一括削除する」** か **「本番用の新しいコレクション（テーブル）に参照先を切り替える」** のいずれかの対応が必要です。

ここでは、最も安全で簡単な **「データの参照先（コレクション名）を切り替える方法」** を推奨します。

---

## 準備編：ソースコードの変更（イベント前日〜当日朝）

現在、アプリは `progress_history` という名前の場所にデータを保存・読み込みしています。これを、本番用の新しい名前（例: `progress_history_production`）に変更します。

### 手順

1. エディタ（VS Codeなど）でプロジェクトを開きます。
2. 以下の2つのファイルを探します。
   - `src/lib/services/history.ts`
   - `src/lib/services/location.ts`

3. それぞれのファイル内で、**`'progress_history'`** という文字列を検索し、**`'progress_history_production'`** （または任意の本番用名前）に書き換えます。

**修正イメージ:**

```typescript
// 変更前
const q = query(collection(db, 'progress_history'), ...);
await addDoc(collection(db, 'progress_history'), ...);

// 変更後
const q = query(collection(db, 'progress_history_production'), ...);
await addDoc(collection(db, 'progress_history_production'), ...);
```

4. 変更を保存し、GitHubの `main` ブランチにプッシュ（デプロイ）します。

---

## 必須作業：Firestoreのインデックス再作成（デプロイ直後）

コレクション名（保存場所）を変更した場合、Firebase側で「他の参加者の最新位置」を高速に検索するための「インデックス（索引）」を新しく作り直す必要があります。これをしないと、先頭や最後尾のピンが表示されません。

### 手順

1. アプリをデプロイした後、自分（またはテスト用のスマホ）でアプリを開き、「現在地を更新」ボタンを押します。
2. パソコンのブラウザで [Firebase Console](https://console.firebase.google.com/) を開きます。
3. 今回のプロジェクト（`tohodan-r189` など）を開き、左メニューから **[Firestore Database]** をクリックします。
4. **[インデックス (Indexes)]** タブを開きます。
5. **「複合 (Composite)」** のセクションで「インデックスを追加」をクリックし、以下のように設定して作成します。
   - コレクション ID: **`progress_history_production`** （先ほどコードで変更した名前）
   - フィールド 1: **`uid`** （昇順 / Ascending）
   - フィールド 2: **`timestamp`** （降順 / Descending）

> **💡 ヒント:** インデックスの作成には数分〜5分程度かかります。ステータスが「構築中」から「有効」に変われば準備完了です。

## （代替案）テストデータを全削除する方法

もしソースコードを書き換えるのが不安な場合は、Firebase Console から直接テストデータを削除することも可能です。

1. [Firebase Console](https://console.firebase.google.com/) > [Firestore Database] > [データ] タブを開きます。
2. 左側のリストから `progress_history` をクリックします。
3. 中央の列に並んでいるドキュメント（ランダムな文字列の行）の右端にある **「︙（3つの点）」** をクリックし、「ドキュメントを削除」を選択します。
4. テストデータが多い場合は、コレクション `progress_history` そのものを削除してください。（削除しても、アプリから新しいデータが送られてくれば自動的に再作成されます。この方法ならインデックスの再作成は不要です。）

## 最終確認（イベント開始前）

以下の確認を必ず行ってください。

- [ ] アプリを開いた際、**地図上に自分以外のピン（先・後など）が表示されていない**こと。
- [ ] 「現在地を更新」を押すと、自分の位置が正しく青いピンで表示されること。
- [ ] （可能であれば別のスマホでも開き）一方が更新した数分後、もう一方の画面に「先」や「後」のピンとして反映されること。

問題がなければ、本番環境の準備は完了です。参加者の皆様にURLを共有してください。
