# プロジェクト固有ルール: 徒歩団参アプリ開発

## 1. 開発の鉄則

### 1.1. ファイル編集時の事前確認（最優先）
ユーザーが直接ソースコードの文言やロジックを修正している場合があるため、AIエージェントは以下の手順を厳守すること。

1.  **必ず編集前に最新のファイル内容を読み込むこと (`cat` や `read_file` を使用)。**
2.  読み込んだ内容と、自分がこれから行おうとしている修正内容を比較すること。
3.  **ユーザーが行った独自修正（文言の変更、スタイルの調整など）を破壊してはならない。**
4.  修正を加える際は、既存のユーザーによる変更を保持したまま、必要な箇所のみをピンポイントでマージ（追記・置換）すること。ファイル全体を自分のテンプレートで上書きすることは厳禁とする。

### 1.2. コミュニケーション
- ユーザーの意図が不明確な場合や、ユーザーの修正箇所と競合しそうな場合は、作業前に必ず確認を行うこと。

## 2. デザイン・UI標準
- スタイリッシュなデザインを維持しつつ、高齢者にも配慮した大きめの文字サイズ、高コントラストな配色を維持すること。
- モード切り替え時のアクセントカラー（青/緑）の使い分けを尊重すること。

## 3. バッテリーへの配慮
- GPS取得やリアルタイム更新など、バッテリーを激しく消耗する処理は最小限に留め、必要に応じてユーザーへの注意喚起を含めること。
